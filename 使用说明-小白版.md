# 使用说明（小白版）

> 这份说明写给完全没用过 AI 编程工具的朋友。跟着做就行，不需要懂代码。

## 这个东西是干什么的？

你给它**一张图**（动画角色截图，或者一张真人照片），它会帮你做出**一整套 Q 版小人的"动作 × 表情"图片**，比如：站着开心、挥手告别、趴着无语、躺着犯困、五体投地大哭……

做出来的每张图都是**透明背景的 PNG**，可以直接拿去做微信/QQ 表情包、直播贴图、PPT 素材、个人头像等。

效果可以直接看本仓库的 [`Sample/成果`](./Sample/成果) 文件夹——里面 31 张图就是用 `Sample/原图.png` 这一张图生成的。

## 使用前需要准备什么？

1. **一个支持 Skill 的 AI 工具**，比如：
   - Claude Code（Anthropic 官方 CLI / 桌面版）
   - OpenAI Codex
   - 其他支持读取 SKILL.md 格式技能包、且**带图片生成能力**的 AI 代理
2. **一张人物图片**，要求：
   - 人物清楚、完整（最好全身或大半身）
   - 遮挡少、不要太糊
   - 一张图里只有一个主角

> ⚠️ 这个 Skill 本身不画图，它是一套"指挥 AI 画图的完整流程规范"。所以你的 AI 工具必须自带图片生成功能。

## 安装（把 Skill 装进你的 AI 工具）

Skill 本体就是 [`Skill/character-storyboard-generator`](./Skill/character-storyboard-generator) 这个文件夹。把它复制到你的 AI 工具的技能目录即可：

- **Claude Code**：复制到你项目的 `.claude/skills/` 目录下（或个人目录 `~/.claude/skills/`），即：
  ```text
  你的项目/.claude/skills/character-storyboard-generator/
  ├─ SKILL.md
  ├─ references/
  ├─ assets/
  ├─ scripts/
  └─ agents/
  ```
- **Codex**：复制到 Codex 的 skills 目录，保持文件夹结构不变。

装好后重启工具，AI 就能识别到这个 Skill 了。

## 开始使用（三步走）

### 第 1 步：把图片给 AI

把你准备的人物图片放进工作目录，或直接拖进对话框。

### 第 2 步：说一句话

对 AI 说：

```text
使用 $character-storyboard-generator，把这张图制作成透明背景的动作表情分镜。
```

也可以加上你的具体要求，例如：

```text
使用 $character-storyboard-generator，把这张图做成 Q 版分镜。
先只要"开心、生气、哭泣、无语"四种表情的站立版本。
```

### 第 3 步：确认校准图，然后等结果

AI 会先生成**少量校准图**给你看，你需要确认：

- 脸型、发型像不像？
- 衣服、颜色对不对？
- Q 版比例可不可爱？

你说"可以"之后，AI 才会继续批量生成剩下的组合。**如果校准图不像，直接说哪里不像，让它重做**——这一步不确认好，后面全套都会歪。

## 生成结果长什么样？

所有成品会放在 `成果/` 文件夹里，按"情绪-表情"分类：

```text
成果/
├─ 喜-开心/
│  ├─ xiaomai__P01-standing-neutral__E01-joy__v01__v1.0.png   ← 站立开心
│  └─ xiaomai__P06-wave-goodbye__E01-joy__v01__v1.0.png       ← 挥手开心
├─ 怒-愤怒/
├─ 哀-哭泣/
└─ 静-无语/
```

文件名规则（不用记，看得懂就行）：

```text
角色名__动作编号-动作__表情编号-表情__版本号__v1.0.png
```

## 能做哪些动作和表情？

- **30 种动作**：站立、立正、叉腰、抱臂、鞠躬、挥手、敬礼、点赞、比心、跳跃欢呼、走路、跑步、蹲、跪、正坐、盘坐、懒趴、仰躺、侧躺、蜷缩、五体投地、扑街、伸懒腰、探头偷看、双手拜托、比耶……
- **40 种表情**：开心、愤怒、哀伤、大笑、震惊、惊吓、委屈、哭泣、无语、平静、困倦、害羞、尴尬、得意、坏笑、疑惑、坚定、吃醋、星星眼……

完整清单见 [poses.json](./Skill/character-storyboard-generator/references/poses.json) 和 [expressions.json](./Skill/character-storyboard-generator/references/expressions.json)。

理论上可交叉组合出 1200 种图（30 × 40）。**建议按需生成**，一次要几张说几张，不必全跑。

## 常见问题

**Q：我打开图片看到黑色/白色背景，是不是失败了？**
A：不一定。很多看图软件会用黑、白或棋盘格来显示透明区域。把图片拖进 PPT 或聊天窗口试试——真正的透明图不会带底色。

**Q：为什么"性感"表情做不了？**
A：这是刻意设计。角色是未成年人、看起来年幼、或年龄无法确认时，Skill 会强制启用未成年人保护，自动停用性感组合并禁止暴露服装、暗示性姿势等。这不能被绕过。

**Q：生成的人物越来越不像原图怎么办？**
A：告诉 AI"人物跑偏了，回到校准参考图重新生成这张"。Skill 规定每张图都要和最初确认的身份参考图保持一致，跑偏属于不合格品。

**Q：可以一次生成全部 1200 张吗？**
A：技术上可以（Skill 支持分批断点续做），但耗时且费钱。建议先做最常用的 20～50 张。

**Q：真人照片安全吗？**
A：Skill 会把真人转成 Q 版卡通形象，只保留发型、脸型、眼镜等辨识特征，不做精确的人脸复刻。请只使用你本人或已获授权的照片。

## 进阶：脚本工具（可选，不用也完全没关系）

`scripts/` 里有两个 Python 脚本，AI 会在流程中自动调用，你一般不需要手动碰：

- `build_manifest.py`：根据项目配置生成"任务清单"（哪个动作配哪个表情、文件叫什么名字）。
- `audit_outputs.py`：检查生成结果有没有缺图、命名错误、尺寸不对、缺少透明通道。

如果你想手动跑，需要安装 Python 3，然后：

```bash
python scripts/build_manifest.py 你的项目/project.json
python scripts/audit_outputs.py 你的项目/project.json --only-existing
```
